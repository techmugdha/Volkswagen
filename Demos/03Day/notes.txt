# Docstrings in python
: These are used to provide documentation for functions, methods, classed and modules.
: They are written as the first statement in the function or class definations and
: they are enclosed in triple quotes(''' Or """).
: These strings helps you and others understand what the function or class is meant to do, and they can be accessed using the buil-in help() function or the __doc__ attribute.

## Difference between parameters and arguments:
# Parameters :-
  - Variables for holding actual values for function
  - Present in function definition.
# Arguments :-
  - Values provided at the time of function call

----------------------------------------------

## Type of arguments:

1. Positional arguments
2. Keyword arguments
3. Default argument / optional parameters
4. Variable length Positional arguments and keyword variable lenth arguments

---------------------------------------------

# Positional arguments:
The order in which you pass the values during the function call should match the order in which the parameters are defined in the function.

no. of parameters = no. of arguments

# Keyword arguments:
We do not need to maintain order
no. of parameters = no. of arguments

----------------------------------------------


Priority rules:
1. Postional argument
2. Variable length Positional arguments
3. Keyword argument
4. Variable length keyword arguments
5. default argumets
